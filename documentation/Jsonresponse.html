responseJson Endpoint

This endpoint is used to return data to the client in the form of a JSON encoded string. It has a `message` and `responseCode` property that can be set, as well as a `headers` method that can be overridden to set custom HTTP headers.

Usage

To use this endpoint, create an instance of the `responseJson` class and use the `setMessage` and `setResponseCode` methods to set the appropriate values. Then, use the `setData` method to set the data that will be returned in the response. Finally, use the `getData` method to get the data as a JSON encoded string.

The `getData` method will automatically handle setting the `message` and `responseCode` based on the contents of the `data` property if they have not been set manually. If the `data` property is an empty array, the `message` will be set to "No content to display" and the `responseCode` will be set to 204 (No Content). If the `data` property is not an empty array, the `message` will be set to "OK" and the `responseCode` will be set to 200 (OK). If the `responseCode` is set to any other value, it will be used as is and the `message` and `path` will be included in the response.
