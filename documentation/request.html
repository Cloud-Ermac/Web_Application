Request API Documentation

This API allows for the processing of HTTP requests made to the server. It contains information about the request method and the request path.

Methods

__construct()

Initializes the Request object by calling the setMethod and setPath methods.

setMethod()

Retrieves the request method from the $_SERVER superglobal and sets it to the $method property.

validateRequestMethod()

If the request method is not in the array of valid methods passed as an argument, an error message is output and the script is terminated.

setPath()

Retrieves the request path from the $_SERVER superglobal, removes the base path, and sets it to the $path property.

getPath()

Returns the request path.

Examples

Initializing a Request object

  $request = new Request();

Validating a request method

  $request->validateRequestMethod(['GET', 'POST']);

Retrieving the request path

  $path = $request->getPath();

Properties

$method

The request method (e.g. GET, POST, PUT, DELETE).

$path

The request path (e.g. '/api/author').

Errors

If the request method is invalid, an error message will be output with the following format:

{
  "message": "Invalid request method: [request method]"
}

Additional Functionality

In addition to the methods and properties listed above, you can also add additional functionality to the Request class to customize it to your specific needs. For example, you may want to add a method to validate the request path or add additional properties to store additional information about the request.
